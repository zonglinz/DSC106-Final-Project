Prototype writeup

Required project description

What have you done so far?

So far, we have converted the proposal idea into a working webpage structure with a hero section, control panel, interactive dashboard, data note, and project writeup. We loaded the proposal-stage CMIP6 city dataset into the page and connected it to multiple rough visualizations: a climate-twin map, a similarity ranking chart, a San Diego warming trajectory, and a seasonal comparison chart. We implemented several working interactions, including scenario buttons, future-period buttons, a year slider, clickable city points, and clickable similarity bars. We also added explanatory text so a viewer can understand why the project uses climate twins rather than only showing abstract temperature changes. This initial prototype is not the final polished design, but it demonstrates the main user flow we plan to refine for the final submission.

What will be the most challenging of your project to design and why?

The most challenging part will be designing a climate-twin comparison that is both scientifically honest and easy for a general audience to interpret. A similarity score can hide many details, so we need to show enough supporting evidence, such as seasonal temperature patterns and precipitation changes, without overwhelming the reader. It will also be difficult to balance the map, ranking chart, line chart, and narrative annotations so that the interface feels like one connected explanation rather than four separate graphics. Another challenge is communicating uncertainty because climate projections depend on scenario choices, model assumptions, time periods, and local variability. We expect the final design work to focus on making those tradeoffs visible while still keeping the main question, “When does San Diego become Phoenix?”, engaging and intuitive.

Data and status

CMIP6 city-level prototype dataset

Dataset used in this prototype

The prototype uses the processed city-level JSON/CSV dataset included with the project proposal. The records summarize selected U.S. city locations, annual San Diego temperature anomalies, seasonal temperature profiles, climate-twin similarity scores, and seasonal precipitation changes. The proposal describes this as a proposal-stage processed dataset with the same schema as the planned Google Cloud / NOAA CMIP6 extraction output.
